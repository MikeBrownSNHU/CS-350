## Selected Artifacts

### Thermostat Final Project

This was the biggest project in the course and brought together most of the concepts we learned. The project used a temperature sensor, LCD display, LEDs, buttons, UART communication, and a state machine to simulate a thermostat system. I selected this artifact because it shows both software development and hardware integration.

### Serial Light Control Project

This project focused on controlling hardware through serial communication. Commands were sent from a client to a server running on the Raspberry Pi, which then controlled an LED. I chose this project because it demonstrates communication between systems and troubleshooting hardware and software together.

---

## Summarize the project and what problem it was solving

The projects in this course focused on learning how software can interact with physical hardware. Through different labs and the final project, I learned how to read data from sensors, control outputs, communicate between devices, and organize program behavior using state machines. The thermostat project solved a simple real-world problem by monitoring temperature and allowing a user to change operating modes and temperature settings.

## What did you do particularly well?

I think troubleshooting was one of my stronger areas throughout the course. Many times the issue wasn't just the code. Sometimes it was wiring, configuration settings, or hardware communication. I got better at breaking problems down into smaller pieces and testing one thing at a time until I found the root cause.

## Where could you improve?

I would like to become more comfortable designing embedded systems completely from scratch. I learned a lot during the course, but I still have room to grow when it comes to low-level hardware programming and designing more complex systems without following a guide.

## What tools and/or resources are you adding to your support network?

Some of the resources I plan to continue using are the Raspberry Pi documentation, Python documentation, GitHub, hardware datasheets, and online forums. I also found that creating diagrams and mapping out state machines before coding helped me understand the overall design much better.

## What skills from this project will be particularly transferable to other projects and/or course work?

The biggest skills I will carry forward are troubleshooting, hardware integration, serial communication, and state machine design. These skills apply not only to embedded systems but also to automation projects and software development in general. Learning how to systematically work through problems is something I use both in school and in my career.

## How did you make this project maintainable, readable, and adaptable?

I tried to keep the code organized by separating functionality into logical sections and using clear variable and function names. Comments were added where they helped explain the purpose of the code. Using a state machine also made the program easier to follow because each mode had a defined purpose and behavior. This approach would make it easier for someone else to understand the project or add new features later.
